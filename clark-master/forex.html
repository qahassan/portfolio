1st Forex Tading Application

Project Overview

The 1st Forex Web and Mobile Application was designed to provide users with a seamless experience in trading, analyzing signals, managing subscriptions, and viewing real-time analytics. As a QA Engineer, I was responsible for testing both the web and mobile platforms to ensure feature stability, performance, and accuracy of trading data.

  • User authentication and role-based access control
  • Subscription module with Stripe integration for payment plans
  • Community module with post, comment, and notification features
  • Advanced analytics dashboard displaying user activities and trading insights

My contributions included testing all frontend modules, validating API responses using Postman and Swagger, and preparing complete QA documentation for limitations and requirements. I collaborated closely with developers to ensure proper API integration and consistent UI behavior across devices.

Challenges & Problem-Solving Approach

One of the primary challenges was dealing with inconsistent API responses during the integration phase. For instance, missing parameters like titles and descriptions in course modules caused frontend rendering issues. Here's how I tackled these:

  • Created detailed API documentation specifying frontend requirements and limitations
  • Flagged mismatched responses early and coordinated with the backend team to fix them before production
  • Ensured Stripe integration worked seamlessly, especially the 14-day trial-to-paid transition

I also led the bug tracking and verification processes, keeping Jira updated and testing all modules, including user portal, admin panel, and mobile views. My hands-on approach helped streamline communication between QA and Dev teams, resulting in fewer regression issues.

User Experience & Functionality Testing

I focused on delivering a flawless user experience through careful validation of all modules, especially Subscription, Analytics, Courses, and Community. This included extensive UI/UX checks and testing data consistency between backend and frontend layers.

  • Verified frontend limitations (e.g., profile info, enrolled members, course progress) were covered in API responses
  • Tested payment flow including Stripe key validations and webhook functionalities
  • Validated the full trial-to-subscription lifecycle and subscription status transitions

I also reviewed mobile app testing for token issues, verified post/comment features in the Community module, and ensured that Notification functionalities were accurate and timely across the platform.

Technical Highlights

The 1st Forex platform leverages modern frontend frameworks with RESTful API backend services. My QA process included Postman testing, documentation for frontend limitations, and support for Swagger-based API testing. Key tech responsibilities:

  • API testing and validation with Postman and Swagger
  • Created Postman collections and shared environment variables for collaborative testing
  • Documented design limitations to help frontend handle missing parameters or UI issues
  • Worked closely with Dev team to resolve token, enum, and response structure errors

My role was instrumental in preparing the application for the first internal demo and production handover. The project benefitted from early bug identification, user-friendly testing reports, and close coordination with cross-functional teams to ensure timely and successful delivery.
